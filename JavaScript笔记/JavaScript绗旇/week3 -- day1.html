Git

命令行工具
分布式版本控制系统
免费的 开源的
通过git

GitHub

项目托管平台
通过Git命令 将代码推送到GitHub保存起来
通过git每次提交都会生成一个版本(历史记录) 可以通过命令行切换版本

git仓库的概念

1.新建一个文件夹
2.在当前目录下打开命令窗口 鼠标右键 -> git bash here
3.把这个文件夹变成git仓库

将本地仓库和远程仓库关联起来

git remote add origin 远程仓库的地址

查看当前仓库(关联远程仓库的列表)

git remote -v

从工作区添加到暂存区

git add -A 将所有的文件添加到暂存区

查看当前的状态

如果文件名称是红色 说明当前文件在工作区
如果文件名称是绿色 说明当前文件在缓存区

配置用户名和邮箱

git config --global user.name '用户名'
git config --global user.email '邮箱名称'

查看提交记录

git log 查看提交记录
git log --online 把提交记录显示一行

推送到远程仓库

git push origin master

git的三个区域

工作区：本地仓库文件夹
暂存区：缓存当前文件
历史区：当文件提交到历史区 就会生成历史记录 会有一个版本号

git 流程

git init --> git remote add origin '仓库地址' --> git add ./-A --> git commit -m '备注信息' --> git push origin master

把已有的项目变成git仓库 推送到远程

第一种方式：
1.git init 把当前本地文件夹 初始化成一个git仓库
2.git add. 把工作区中的所有文件 添加到 暂存区
3.git commit -m '此次提交的描述信息' 将暂存区的文件提交到历史区 会生成一个版本号
4.git push origin master 将历史区的文件 推送到远程仓库的主干
创建一个远程仓库地址 建立连接：

  • git remote add origin <远程仓库地址>
  • git remote -v 查看远程仓库链接列表
    第二种方式：
    1.现在gitHub上创建一个 远程仓库 拉取到本地
    2.git clone <仓库地址> 克隆仓库上面的文件
    本地仓库会默认和远程仓库建立联系
    3.git pull origin master 拉取并且合并远程的代码到本地

linux命令

创建文件夹 mkdir 文件夹名称
创建文件 touch 文件名.后缀名
编辑,写入内容

  • echo hello > index.txt 将hello写入index.txt
  • echo hell >> index.txt 追加内容到index.txt

查看文件内容 cat 文件夹名称
切换目录

  • cd 文件夹名称 进入到某个文件夹内
  • cd .. 回到上一级目录

移动文件 mv 1.js a/j.js
