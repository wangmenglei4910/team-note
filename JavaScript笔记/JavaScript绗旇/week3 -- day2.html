JS引擎

就是用来解决执行JS代码的
JS引擎分为2个步骤

  • 1.解释：全盘扫描当前作用域中的代码 变量提升就是发生在解释阶段
  • 2.执行：从上到下执行JS代码

变量提升(预解释)

在作用域形成之后，代码执行之前 把var和function声明的变量 进行提前声明和定义
对于带var提前声明 并且赋值为默认值 undefined
对于带function 提前声明 并且赋值为函数

  • 变量提升 只发生在当前作用域
  • 最先开始 全局作用域中进行变量提升
  • 函数执行的时候 函数作用域进行变量提升

变量提升优先级
函数变量提升 > 形参变量提升 > 变量提升

变量提升的特殊情况：
  • 等号右边不会进行变量提升
  • 自执行函数不会进行变量提升
  • return后面的不会进行变量提升，return下面的会进行变量提升
  • 函数当做参数的时候不会变量提升
  • 在碰到块级作用域的时候,函数只声明不定义；比如判断，当条件一旦成立，第一时间对函数进行定义
带var和不带var的区别：
  • 通过变量名声明的变量，既是在当前作用域下声明的一个变量，也是给全局对象window定义一个对象
  • 给window全局对象赋值，'window.'可以忽略不写
  • 不带var的不会进行变量提升
  • delete 删除不了用var声明的变量
  • 可以删除不带var添加的变量
闭包

在一个函数里 返回一个内部函数 这个内部函数可以访问上级作用域的变量 这个内部函数就称之为闭包
闭包是一种机制 可以将变量缓存

作用域,作用域链

作用域：全局作用域 函数作用域 块级作用域
作用域链：在一个作用域中查找变量，首先在当前作用域中查找，如果当前作用域中没有 就去上一级作用域中查找 一直查找到全局作用域 如果全局作用域中也没有要查找的变量 就直接返回 '变量' is not defined (在作用域链查找的过程中 如果查到window也没有这个属性 就相当于给window设置了这个属性)

let 和 const
  • 1.let const 声明的变量不会给window增添属性
  • 2.let const 声明的变量不会进行变量提升
  • 3.let const 声明的变量不允许重复声明
  • 4.let const在{}中声明变量会形成块级作用域

私有作用域中带var和不带var的区别：

1，带var的在私有作用域中会有变量提升 都声明为私有变量 和外界没有关系
2，不带var的就不是私有变量 会向它的上级作用域查找 看是否为上级作用域的变量 如果不是 继续向上级查找 一直找到window (我们把这种查找机制成为作用域链)，也就是我们在私有作用域中操作的这个非私有变量 操作的是别人的变量
